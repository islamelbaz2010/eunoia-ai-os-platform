# BEFORE / AFTER ONBOARDING

**Date**: 2026-07-07  
**Task**: Task 2 — First five minutes premium experience

## Landing Page

| Before | After |
|---|---|
| Primary CTA pushed visitors toward booking a demo. | Primary CTA now supports immediate signup with `Start Free Trial` / `Start Free`. |
| Sticky CTA repeated demo-first behavior. | Sticky CTA now invites users to start immediately while preserving demo intent elsewhere. |

## Sign Up

| Before | After |
|---|---|
| Centered form with little context. | Premium two-column desktop layout with product promise and setup steps. |
| Button said `Creating account...`. | Button says `Creating secure account...`. |
| Plain red error text. | Styled error panel with clearer visual hierarchy. |
| No expectation for what happens next. | Copy explains that workspace creation follows account creation. |

## Sign In

| Before | After |
|---|---|
| Centered form only. | Product context beside the form on desktop. |
| Button said `Signing in...`. | Button says `Opening workspace...`. |
| Plain red error text. | Styled error panel. |
| Generic support copy. | Copy explains the active organization opens automatically. |

## Organization Creation

| Before | After |
|---|---|
| Single form with minimal explanation. | Guided `Step 2 of 2` setup screen. |
| Generic workspace setup copy. | Explains organization-scoped knowledge, CRM, usage, and audit logs. |
| Button said `Creating workspace...`. | Button says `Preparing dashboard...`. |

## First Dashboard

| Before | After |
|---|---|
| Empty dashboard showed zero KPI cards and blank chart text. | Empty dashboard now includes first-run guidance and a `First five minutes` checklist. |
| Greeting was generic. | Heading references the active organization and gives next-step guidance. |
| No quick actions. | Quick actions link to `Ask AI` and `Add knowledge`. |
| Empty charts said `No usage data yet` and `No contacts yet`. | Empty panels explain what creates data and provide CTAs. |

## Empty CRM

| Before | After |
|---|---|
| Empty row said to add the first contact above. | Rich empty state explains what kind of contact to add and why. |
| Search/filter empty states were plain text. | Search, archived, and deleted views now have specific messages and filter-clearing path. |
| Quick add button was minimal. | Quick add button explains accepted contact types and gives clearer feedback. |

## Empty Knowledge Base

| Before | After |
|---|---|
| Form placeholders were generic. | Form explains what source material to paste and how to improve answer quality. |
| No success feedback after embedding. | Success toast confirms the document was embedded. |
| Empty table said `No documents yet`. | Empty state explains that the assistant needs its first trusted source. |

## Empty Analytics / Usage

| Before | After |
|---|---|
| Empty usage page showed `No usage recorded yet`. | Empty usage page explains which first actions will create events. |
| No recovery path from empty analytics. | CTAs route users to ask AI or add knowledge. |

## Loading and Error States

| Before | After |
|---|---|
| Loading skeletons were sparse and did not match page shape. | Loading skeletons now mirror headings, cards, filters, and chart areas. |
| Error states were short and generic. | Error states are reassuring, specific, and include reload actions. |
